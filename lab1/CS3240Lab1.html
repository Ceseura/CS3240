Using assert in Python to check for errors is simple.  It raises an exception if the boolean expression after the keyword is false. Optionally, you can put a string as a 2nd value to be printed as a message.  For example:  assert 3==1,"three not equal to one"

Make sure your testing includes these cases:
  1. Test for n = 0 and for n = 1.  The special case 0! is defined to have value 1.
  2. Test for n = 5.
  3. Test for n being negative.  (What should your code do? Raise the exception ValueError.  How can you test for this?  See https://docs.python.org/3.6/tutorial/errors.html about how to use try/except with exceptions.  You can "throw" an exception using the command "raise", also described on that page.)

Activity 2:

Write Python methods that correctly implement the following functions that manipulate lists.  Put all of these in the file that is listed for each part below.  Your file may contain other methods than the ones specified here. We may run-test the methods listed here.  Follow the specifications carefully. Name each method exactly as listed here.

Put the following functions in a file called lab1_lists.py. (We might use this as a module, and module names must have underscores not hyphens in their names. Also, use the technique for making this a module by which some main() or test methods are only executed if __name__ == "__main__".)

  • maxmin(list)
    Assume the parameter is a any valid list containing comparable values of the same type. Return a tuple where the first value is the maximum value found in the list (based on the > operator) and the second value in the tuple is the minimum value found in the list (based on the < operator).  If the list is empty, return the special Python value None.  Examples:
    For the list [1, 3, 3], return (3, 1)
    For the list [3, 1, -2], return (3, -2)
    For the list ['Q', 'Z', 'C', 'A'], return ('Z', 'A')
    Note: Constraint for this assignment: do not use the built-in Python methods max() or min() for this.  You cannot sort the list either.

  • common_items(list1, list2)
    Assume that list1 and list2 are valid lists.  Return a list that contains items that are found in both lists.  If there are no such common items, return an empty list.  The list returned will not contain any duplicate items.
    Note: Constraint for this assignment: do not use the built-in Python support for sets for this.
